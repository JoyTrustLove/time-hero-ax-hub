AI 복기 프로토콜 (v1)

AI가 준 답을 바둑 복기처럼 되짚어 한 단계 끌어올리는 방법

이 파일을 쓰는 법
AI 챗봇(Gemini·ChatGPT·Claude 등)에 이 파일을 첨부한 뒤, 이렇게 말하세요:
  • "이 지침의 속기로 [주제]를 분석해줘"
  • "이 지침의 정석으로 [주제]를 분석해줘"
  • "이 지침의 끝판왕으로 [주제]를 분석해줘"

→ AI가 아래 절차대로 스스로 복기합니다. 프롬프트를 따로 복사할 필요가 없습니다.

"신의 한수는 두는 자가 못 알아본다. 다음 날 다른 기사가 복기에서 알아본다."
AI에게 한 번 묻고 받은 답은 무난한 '첫 수'일 뿐입니다. 복기하면 한 단계 좋아집니다.

0. AI가 항상 지킬 공통 규칙

  1. 점수·확신은 참고용. 최종 결정은 사용자가 한다. 확신이 약하거나 출처가 불분명한 곳은 반드시 [확인 필요]로 표시한다.
  2. 사용자가 자료를 주면 요약하지 말고 원문 그대로 먼저 읽는다.
  3. 직전 대화에 휘둘리지 않는다. 지금 주제와 받은 자료만 근거로 둔다.
  4. 없는 사실·출처·숫자를 지어내지 않는다.

레벨 1 — 속기 (빠른 판단·메일·짧은 기획)

(속으로 진행) 1) 보통 답 → 2) 자기비판(약한 근거·놓친 관점·반대 의견) → 3) 다른 각도로 치우침 점검 → 4) 종합해 더 나은 답.

(출력) 최종 결론(한 줄) / 핵심 근거(2~3) / [확인 필요] / 한 줄 권고(결정은 사용자).

레벨 2 — 정석 (깊게 → 넓게 → 복기)

세 단계를 차례로, 각 단계는 독립적으로(앞 단계에 휘둘리지 않게). 가능하면 각 단계를 새 대화에서.

1단계 · 깊게 — 한 길을 바닥까지

  1. 보통 답을 낸다.
  2. 거울 던지기: 그 답을 비판(약한 근거·헐거운 논리·뻔한 곳).
  3. 자가진단 3문: ①대충 넘기고 싶은가(그럼 반드시 한다) ②진짜 만족하나(어디서 타협했나) ③회피한 게 있나(그게 핵심).
  4. 두 번 다듬기(84점→92점 목표).
  5. 숨은 핵심 한 가지(신의 한수)를 찾는다.

정리: 핵심 결론 / 근거(3) / 숨은 핵심(1) / 불확실(1~2)

2단계 · 넓게 — 판 전체 둘러보기

깊게의 답은 잠시 잊고 폭넓게 모은다.
6축 점검: 커버리지·다양성·관련성·출처 신뢰도·실전성·미탐색 감각
4편향 점검: 도메인·언어·시대·규모(한쪽으로 치우치지 않았나)

정리: 패턴·사례(4~5, 출처) / 못 살핀 영역(1) / 6축 점수 / 4편향 결과

3단계 · 복기 — 두 답을 견줘 판정

깊게·넓게를 나란히 놓고(새로 찾지 않음), 각 답의 숨은 전제를 적어 서로 반박 가능한지 본다.
4체크(각 0~10): 재발명 · 빈틈 · 모순 · 시너지
판정: 32~40 종합 보고 / 빈틈 ≤5 깊게 다시 / 재발명 ≤5·편향 2개↑ 넓게 다시 / 20 미만 주제 재정의

정리: 종합 결론 / 이유(3) / [확인 필요] / 한 줄 권고(결정은 사용자)

레벨 3 — 끝판왕 (전략·큰 의사결정)

정석의 '깊게'를 9수로 더 깊이 둔다:

  1. 첫 수 — 보통 답(스스로 55점)
  2. 거울 던지기 — 비판
  3. 자가진단 3문
  4. 두 번 다듬기(84→92)
  5. 사활 판단 — 쓸 만한가
  6. 숨은 묘수(78수) — 못 본 한 수 (평범↔명국의 갈림)
  7. 제3자의 눈 — 심판처럼
  8. 고수의 한 수
  9. 착지 — 정리

+ 입회인 채점: 점수를 새 대화로 갈아 다시 채점(자기편들기 차단).
+ 외부 복기(옵션): 다른 AI·사람에게 보여 재확인.
+ 게이트: 근거 출처 2곳 / 자기 규칙 점검 / "졌다 치고" 패인 미리 복기.

출력 형식 (공통 마무리)

  • 핵심 결론 (한 줄)
  • 핵심 근거 (2~3가지)
  • 직접 확인이 필요한 부분 ([확인 필요], 없으면 '없음')
  • 한 줄 권고 — 단, 최종 결정은 사용자가 합니다.

이 지침은 'AI 답을 바둑 복기처럼 되짚어 한 단계 끌어올리는' 방법을 담았습니다. 점수는 참고용이며, 마지막 판단은 사람의 몫입니다.
